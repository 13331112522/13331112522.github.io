What is Claude Code?

Claude Code 101 系列教程 · 第 1 课 · 来自 Claude 官方频道

什么是 Claude Code？

Claude Code 是一个 Agentic 编码工具——它能理解你的代码库、编辑文件、执行命令，并与你现有的开发工具集成，帮助你更快完成工作。

它可在以下环境使用：

  • 终端（Terminal）
  • Visual Studio Code
  • Claude 桌面应用
  • Web 版
  • JetBrains IDEs

本课程以终端使用为主。

Claude Code vs Claude AI：核心区别

如果你用过 Claude AI（网页聊天），你可能会好奇两者的不同。

特性 Claude AI Claude Code
访问方式 网页对话 直接访问本地文件
代码操作 复制粘贴 自动读写编辑
工作模式 聊天助手 AI Agent
终端集成 无 完整终端访问
代码库理解 受限于上下文 深度理解整个 codebase

最直观的区别：Claude Code 以 AI Agent 的方式工作，无需你手动在两个窗口间搬运代码。

什么是 AI Agent？

AI Agent 是一种能够与环境交互、执行操作来完成既定目标的软件。其基本原理是：

  1. 大语言模型实时运行在循环中（LLM in a loop）
  2. Agent 拥有工具访问权限——终端、文件系统、外部服务
  3. 可以调用其他 AI Agent 协作完成目标
  4. 根据中间结果自主决定下一步操作

Claude Code 能做什么？

1. 读取并理解你的代码库

你可以让 Claude Code 解释某个功能的实现，或追踪一个 bug 在代码中的传播路径。

2. 执行构建和测试

  • 运行构建脚本（build script）
  • 执行测试套件（test suite）
  • 安装依赖包
  • 根据输出结果自动决定下一步

3. 搜索网络

当需要查阅最新的 API 文档或参考资料时，Claude Code 可以帮你搜索。

三个关键概念

要高效使用 Claude Code，需要理解以下三点：

1. 上下文窗口（Context Window）

上下文窗口是 Claude 的”工作记忆”——它能容纳大量信息，但不是无限的。

Agentic 的核心优势在于：以策略性的方式在代码库中查找答案，而不是把整个代码库塞进上下文。

2. 权限确认（Permission）

默认情况下，Claude Code 在执行命令或修改代码前会先请求你的许可。你始终拥有控制权——可以选择事必躬亲，也可以让它自主运行。

3. 可能犯错（Not Perfect）

和任何工具一样，Claude Code 并非完美。它可能：

  • 误解你的意图
  • 引入新的 bug
  • 过度工程化一个解决方案

快速回顾

Claude Code 是一个 Agentic 编码工具。它：

  1. 读取你的代码库
  2. 编辑你的文件
  3. 执行命令
  4. 连接外部工具

帮助你更快地交付代码。
