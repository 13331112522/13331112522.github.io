Context Management in Claude Code

Claude Code 101 系列教程 · 第 7 课 · 来自 Claude 官方频道

Context 是 Claude 的工作记忆

Claude Code 中的一切都会占用上下文窗口：

  • 每个被读取的文件
  • 每条运行的命令
  • 每条你发的消息

上下文窗口是有限的，因此优化上下文空间至关重要。

Compaction（自动压缩）

当你接近上下文窗口上限时，Claude Code 会自动 Compact：

  • 总结重要细节
  • 移除不必要的工具调用结果
  • 释放大量上下文空间

注意： 压缩可能会丢失之前对话中的部分细节。

手动压缩

你也可以主动运行 /compact 命令——这在你想释放上下文空间但保留之前工作记忆时很有用。

/clear — 完全重置

如果你想要从零开始、不保留任何记忆，运行 /clear：

  • 清除所有上下文
  • 从头开始

/context — 查看上下文状态

运行 /context 命令可以看到：

  • 当前上下文大小
  • 各类别占用最多上下文的部分
  • 可视化图表展示

何时用 Compact vs Clear

场景 使用命令 原因
正在开发某个功能，上下文接近上限 /compact 保留与当前功能相关的上下文
完成了一个计划，要开始新功能 /clear 避免之前的对话对新功能产生偏见
跨会话需要记住的东西 CLAUDE.md 不用每次重新发现

优化上下文的技巧

1. Prompt 要具体

反直觉的事实：写短的 Prompt 在长期会消耗更多上下文。

因为不够明确的 Prompt 意味着 Claude 必须自己花更多时间去探索代码库、自己做推理——这些都要占用上下文空间。多写一两句说明，反而能节省大量空间。

2. 精简 MCP 服务器

MCP 服务器默认会加载所有可用工具到上下文中。如果你有很多与当前项目无关的 MCP 服务器，考虑关掉它们。

替代方案： 尝试 Skills——与 MCP 类似但不会把整个内容放入上下文，帮你节省空间。

3. 使用 Sub-agents

Sub-agents 与主 Agent 并行运行，拥有完全独立的上下文窗口。

适合场景：

  • 只需要答案、不需要过程
  • 例如：“认证端点在哪里？”

Sub-agent 完成工作后，只向主 Agent 返回摘要。

总结

  • /compact — 总结长会话，保留关键上下文
  • /clear — 完全从零开始
  • /context — 检查上下文使用情况
  • Prompt 要具体 — 短 Prompt 反而更费上下文
  • 精简 MCP — 关掉不相关的服务器
  • Sub-agents — 只需要答案的任务交给子代理
