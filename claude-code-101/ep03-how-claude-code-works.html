How Claude Code Works

Claude Code 101 系列教程 · 第 3 课 · 来自 Claude 官方频道

The Agentic Loop

Claude Code 的核心是 Agentic Loop（代理循环）。理解这个循环，Claude Code 就不再是一个”黑盒”。

循环的四步

  1. 你输入一个 Prompt — 告诉 Claude Code 你要做什么
  2. 收集上下文（Gather Context） — Claude Code 与模型交互，模型返回文本或工具调用
  3. 执行操作（Take Action） — 例如编辑文件、运行命令
  4. 验证结果（Verify Results） — 判断是否达成了 Prompt 的目标

如果验证通过，Claude Code 完成任务，等待下一个 Prompt。 如果验证不通过，Claude Code 回到循环顶部重新执行，直到结果完整且可验证。

在整个循环中，你可以随时添加上下文、中断操作、或引导模型朝向你的目标。

Context Window（上下文窗口）

上下文窗口决定了 Claude 能存储多少信息——包括对话内容、文件内容、命令输出等。

当你接近上限时，Claude Code 会自动 Compact（压缩）：

  • 保留重要细节
  • 移除不必要的工具调用结果
  • 释放上下文窗口空间

注意： 压缩可能会丢失之前对话中的部分细节。

Tools（工具）

工具是 Agent 工作方式的基石。

传统 AI 助手是：输入文本 → 输出文本，中间什么都没有。

Claude Code 通过工具实现真正的操作：

  • Read File Tool — 读取项目文件
  • Search Web Tool — 搜索网络文档
  • Run Command Tool — 执行终端命令
  • Edit File Tool — 编辑代码文件

Claude Code 使用 语义搜索（Semantic Search） 来决定何时调用哪个工具，并获取其输出。

Permission Modes（权限模式）

Claude Code 有多种权限模式，默认行为是每次编辑文件或运行命令前都要显式请求你的许可。

切换方式

使用 Shift + Tab 在不同模式间切换：

模式 行为
默认模式 编辑文件和运行命令前都要询问许可
Auto Accept 自动批准文件编辑，但仍需许可才能运行命令
Plan Mode 只使用只读工具，先分析再制定行动计划

建议： 跳过权限时要谨慎。给 Claude Code 自由运行命令的权限意味着错误可能更难被及时发现。

总结

Claude Code 将多个 Agentic 概念组合在一起：

  1. Agentic Loop — 收集 → 执行 → 验证 → 循环
  2. Managed Context Window — 自动压缩，保持上下文精简
  3. Tools — 从”只聊天”到”真正操作”
  4. Configurable Permissions — 你决定 Claude Code 有多少自由度

它能够读取你的代码库、执行操作、验证自己的工作——这是它与普通聊天窗口的根本区别。
