The CLAUDE.md File

Claude Code 101 系列教程 · 第 5 课 · 来自 Claude 官方频道

什么是 CLAUDE.md？

CLAUDE.md 是 Claude Code 最有用的功能之一——它给 Claude Code 持久化的项目记忆。

没有 CLAUDE.md，每次启动 Claude Code 都要从头开始：

  • 重新探索代码库
  • 重新理解依赖关系
  • 重新了解已有功能
  • 有时还要猜测——让引导变得更困难

有了 CLAUDE.md，Claude Code 每次启动时自动读取，就像一份代码库的入职手册。

简单来说： CLAUDE.md 的内容会被追加到你的每个 Prompt 中。

如何创建

运行 /init 命令，Claude 会根据你的代码库自动生成一份 CLAUDE.md。

示例内容

一个典型的 CLAUDE.md 可能长这样：

This is a Next.js 15 app using the App Router, Tailwind, and Drizzle ORM.

Commands:
- dev server: npm run dev
- run tests: npm test
- lint: npm run lint

Code style:
- Use two-space indentation
- Prefer named exports
- All API routes go in app/api
- Use server actions instead of API routes where possible

有了这些信息，当你让 Claude Code 创建一个 React 组件时，它立刻知道该用 Tailwind 样式，而不用先花时间摸索你的 CSS 方案。

文件层级

CLAUDE.md 有一套记忆文件层级，根据面向的用户不同：

层级 位置 作用
项目级 项目根目录 CLAUDE.md 团队共享，提交到版本控制
用户级 ~/.claude/CLAUDE.md 仅你自己，跨所有项目，放个人偏好

三个实用技巧

1. 让 Claude 保存记忆

如果你不得不纠正 Claude 的行为（比如”总是用 server actions 而不是 API routes”），让 Claude 把这条规则保存到 CLAUDE.md。下次回到项目时，它会自动记住。

2. 引用项目文档

使用 @ 符号加上文件路径，让 Claude 引用项目中的文档：

@docs/architecture.md

3. 从空白开始

推荐先不带 CLAUDE.md 开始项目，观察哪些地方需要反复纠正。这样你的 CLAUDE.md 会保持精简，只包含 Claude 真正需要的必要信息。

总结

一次令人沮丧的 Claude Code 会话和一次高效的会话之间的区别，就在于上下文——而 CLAUDE.md 就是你提供上下文的方式。

从你的技术栈、偏好、命令开始，然后随着使用逐步完善。
